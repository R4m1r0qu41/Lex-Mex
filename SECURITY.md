# Security policy

## Supported versions

Lex-Mex is currently pre-release software. Security fixes are applied to the
latest commit on `main`; no older release line is supported yet.

## Reporting a vulnerability

Do not open a public issue for a vulnerability that could expose credentials,
corrupt canonical legal data, bypass provenance checks, or falsify review
state. Report it privately through
[GitHub Security Advisories](https://github.com/R4m1r0qu41/Lex-Mex/security/advisories/new).

Include the affected revision, reproduction steps, impact, and any suggested
mitigation. Do not include real secrets or sensitive third-party data.

Ordinary correctness bugs, including parser defects that do not create a
security boundary failure, may be reported through public issues.
